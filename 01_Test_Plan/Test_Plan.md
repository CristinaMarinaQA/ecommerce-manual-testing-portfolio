## 1. Introduction

This document describes the testing strategy and scope for the demo e-commerce web application: https://automationexercise.com

The purpose of this project is to simulate a real-life QA process and demonstrate structured manual testing documentation.

---

## 2. Objectives

The objective of testing is to:

- Verify that core e-commerce functionalities work as expected
- Identify functional defects and usability issues
- Validate input fields and error handling
- Ensure a smooth end-to-end user journey

---

## 3. Scope of Testing

### In Scope

The following modules will be tested:

- User Registration
- User Login / Logout
- Product Browsing
- Product Details Page
- Add to Cart
- Cart Management
- Checkout Process
- Contact Form
- Form Validations

### Out of Scope

- Performance Testing
- Security Testing
- Payment Gateway real transaction validation
- Mobile application testing

---

## 4. Test Types

The following testing types will be applied:

- Functional Testing
- Negative Testing
- Boundary Value Analysis
- Exploratory Testing
- Usability Testing

---

## 5. Test Environment

- Application URL: https://automationexercise.com
- Browser: Google Chrome (latest version)
- Operating System: Windows 11
- Testing Type: Manual Testing

---

## 6. Entry Criteria

- Application is accessible
- Test data is available
- Browser is properly installed

---

## 7. Exit Criteria

- All planned test cases are executed
- Critical and high-severity defects are documented
- Test Execution Report is completed

---

## 8. Risks

- Demo environment instability
- Limited access to the backend database
- No control over production data

---

## 9. Deliverables

The following documents will be delivered:

- Test Plan
- Test Scenarios
- Test Cases
- Bug Reports
- Test Execution Report
